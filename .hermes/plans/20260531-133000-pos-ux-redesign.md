# POS UX Redesign Plan

Goal: Transform POS (kasir) jadi pengalaman kasir laundry yang cepat, intuitif, minim klik —  
mirip aplikasi POS modern (Moka, Qasir) tapi untuk laundry.

Tanggal: 2026-05-31  
Status: Draft

---

## 1. Context

### Current state
- Layout: 3-col grid pada desktop (`lg:grid-cols-5`, kiri 3 kanan 2)
- Customer: dropdown select + checkbox "pelanggan baru" — tidak optimal untuk >20 customer
- Layanan: grid 2-kolom, semua layanan rata tanpa kategori (cuci kering, setrika, dll)
- Submit: `fetch()` manual, full page reload implicit saat sukses
- Ringkasan: hanya tampil di kanan, tidak interaktif
- Pembayaran: **tidak ada** (padahal tabel `payments` sudah ada di schema)
- After submit: full-page blocking success card
- Tidak ada: pencarian layanan, keyboard shortcut, tombol cepat, cetak struk

### Tech constraints
- SvelteKit 2 + Svelte 5 (runes: `$state`, `$derived`, `$props`)
- Tailwind CSS
- Supabase backend — `payments` table ready (metode bayar: tunai, transfer, gopay, ovo, dana, shopeepay, qris)
- `order_status_log` table ready for tracking
- Schema: `orders.jalur` (antar_sendiri | jemput), `orders.waktu_bayar` (awal | akhir)

### Navigation: top bar, bukan sidebar

POS adalah halaman **high-frequency**. Sidebar 224px memakan ruang yang seharusnya untuk input.  
Alternatif: **collapse sidebar di halaman POS**, ganti dengan top bar minimal.

### Key UX improvements

| # | Improvement | Detail |
|---|-------------|--------|
| 1 | **Search customer** | Input text search by nama/nomor HP, auto-suggest dropdown. Tidak perlu scroll select |
| 2 | **Quick new customer** | Form pelanggan baru inline, tidak toggle checkbox |
| 3 | **Service search** | Input filter layanan, keyboard shortcut (typing langsung scroll ke match) |
| 4 | **Cart panel** | Panel kanan dedicated: list item + qty editor + subtotal + diskon + total |
| 5 | **Pembayaran built-in** | Pilih metode bayar, input nominal, hitung kembalian. **Simpan ke tabel payments** |
| 6 | **Keyboard shortcuts** | Enter = submit, Esc = clear, angka 1-9 = quick-add layanan |
| 7 | **Success flow** | Slide-in toast (bukan blocking card), auto-reset form, fokus ke input customer |
| 8 | **Service categories** | Tag filter: Semua / Cuci / Setrika / Express / dll |
| 9 | **Diskon field** | Input nominal atau persen di cart panel |
| 10 | **Collapse sidebar** | Di halaman POS, sidebar auto-collapse ke ikon saja (atau hide total) |
| 11 | **Recent customer quick-select** | Tampilkan 5 customer terakhir di bawah input |
| 12 | **Waktu estimasi** | Tampilkan estimasi selesai berdasarkan layanan terpilih |

## 3. Step-by-step Plan

### Phase 1: Foundation (layout + cart panel)
**Files:** `(app)/[slug]/pos/+page.svelte`, `(app)/+layout.svelte`

1. **Collapsible sidebar untuk POS**
   - Deteksi route `/pos` di `(app)/+layout.svelte`, auto-collapse sidebar ke 56px (ikon only)
   - Atau: buat `(app)/+layout.svelte` accept prop/slot untuk `fullWidth`

2. **2-panel layout POS**
   - Left: scrollable input area (customer + services + options)
   - Right: sticky cart panel (300px-380px width)
   - Top bar: title + live subtotal

3. **Cart panel component**
   - Item list dengan +/- qty dan tombol hapus (×)
   - Subtotal, diskon (input), total
   - Tombol simpan di bagian bawah

### Phase 2: Customer input
**Files:** `(app)/[slug]/pos/+page.svelte`

4. **Search customer component**
   - Input text dengan debounce search (nama/nomor HP)
   - Dropdown hasil search (maks 5 item)
   - Klik item → pilih customer, tutup dropdown

5. **Inline new customer form**
   - Tombol "+ Pelanggan Baru" di samping search input
   - Klik → expand form inline (nama, nomor HP)
   - Selesai isi → collapse, data tersimpan sebagai selected customer

6. **Recent customers**
   - Query 5 customer terbaru dari server
   - Tampilkan sebagai chip/button di bawah search input
   - Klik langsung pilih

### Phase 3: Service picker
**Files:** `(app)/[slug]/pos/+page.svelte`

7. **Service search/filter**
   - Input filter di atas grid layanan
   - Filter real-time saat mengetik

8. **Category tabs**
   - Query distinct categories atau hardcode: Cuci, Setrika, Express, Lainnya
   - Tab horizontal, klik filter

9. **Service cards UX polish**
   - Tambah icon per kategori
   - Animasi ringan saat qty berubah (scale bounce)
   - Highlight selected items lebih jelas
   - Long-press atau klik kanan untuk quick-add qty >1

10. **Keyboard shortcuts**
    - Ketik nama layanan → auto-select
    - Enter di input qty → add to cart
    - Esc → clear search

### Phase 4: Payment
**Files:** `(app)/[slug]/pos/+page.server.ts`, `(app)/[slug]/pos/+page.svelte`

11. **Payment method selector**
    - Grid tombol metode: Tunai, QRIS, Transfer, GoPay, OVO, Dana, ShopeePay
    - Default: Tunai
    - Setiap metode punya ikon

12. **Payment amount input**
    - Input nominal bayar
    - Auto-hitung kembalian: `bayar - total`
    - Validasi: bayar >= total (atau bisa cicil untuk `waktu_bayar='akhir'`)

13. **Waktu bayar toggle**
    - "Bayar Sekarang" (default) vs "Bayar Nanti"
    - Bayar Nanti → tidak perlu input nominal

14. **Server action update**
    - Insert ke `payments` table
    - Set `orders.waktu_bayar` = 'awal' atau 'akhir'
    - Jika lunas → set `payments.status = 'lunas'`

### Phase 5: Submit + success
**Files:** `(app)/[slug]/pos/+page.svelte`

15. **use:enhance untuk form**
    - Replace `fetch()` manual dengan SvelteKit `enhance`
    - Tidak ada full page reload

16. **Success toast**
    - Toast hijau slide-in dari atas kanan
    - Tampilkan nomor order + total
    - Tombol: "Cetak Struk" + "Tutup"
    - Auto-dismiss setelah 5 detik

17. **Quick receipt view**
    - Modal atau slide-in panel dengan format struk
    - Bisa di-print via `window.print()` (CSS print stylesheet)
    - Tombol "Bagikan via WhatsApp" (opsional)

18. **Auto-reset form**
    - Clear semua input setelah sukses
    - Fokus ke input customer
    - Recent orders table auto-refresh (atau invalidate)

### Phase 6: Polish
19. **Loading states**
    - Skeleton saat load pelanggan/layanan
    - Spinner di tombol submit
    - Disabled state yang jelas

20. **Empty states**
    - "Belum ada layanan" → link ke Pengaturan
    - "Belum ada pelanggan" → form pelanggan baru auto-expand

21. **Error handling**
    - Toast merah untuk error (network, server)
    - Field-level error highlight

22. **Mobile responsive**
    - Single column pada mobile
    - Cart panel jadi bottom sheet (swipe up)
    - Service grid 3-kolom compact

## 6. Risks & Tradeoffs

| Risk | Mitigation |
|------|-----------|
| Sidebar collapse UX | Buat toggle arrow di sidebar, user bisa expand/collapse manual |
| Search customer performance | Debounce 300ms, batasi hasil 5 |
| Payment complexity | Mulai dengan "Tunai" saja, metode lain opsional |
| Keyboard shortcut conflict | Only active when POS is focused, not interfering with browser shortcuts |
| Print styling | Sediakan CSS `@media print` terpisah |

## 7. Open Questions

1. **Sidebar approach:** Collapse total ke 56px icons, atau hide total dan ganti dengan top bar?  
   → Rekomendasi: top bar minimal + sidebar 56px icons-only (mirip aplikasi POS iPad)

2. **Cetak struk:** Perlu ke hardware thermal printer, atau cukup `window.print()`?  
   → Mulai dengan `window.print()` + CSS print, thermal printer nanti

3. **Kategori layanan:** Hardcode atau query dari DB?  
   → Hardcode dulu: Cuci, Setrika, Express, Lainnya. Nanti bisa dari DB kalau dibutuhkan

4. **Multiple payment methods per order?**  
   → Untuk v1, satu metode per order. Split payment nanti
