# Plan: Filter Laporan — Preset + Custom Range + Navigasi

**Date:** 2026-06-01
**Target:** `/Users/inurdiansyah/project/laudryin/src/routes/(app)/[slug]/reports/`

---

## Problem

Saat ini filter laporan hanya 3 tombol (Hari Ini / Minggu Ini / Bulan Ini) yang:

1. **Tidak scalable** — tidak bisa custom range, tidak ada kemarin/bulan lalu/tahun ini
2. **Tidak informatif** — label "2026-06-01" tidak deskriptif; "Minggu ini" tidak menunjukkan tanggal
3. **Tidak bisa navigasi** — tidak bisa mundur ke periode sebelumnya tanpa ganti manual di URL
4. **Mobile cramped** — 3 tombol horizontal, nggak ada ruang untuk lebih

---

## Proposed Design

### 1. Preset Chips (horizontal scroll)

```
[Hari Ini] [Kemarin] [Minggu Ini] [Bulan Ini] [Bulan Lalu] [Tahun Ini] [Custom]
```

- Scrollable horizontal di mobile (`overflow-x-auto`)
- Chip aktif: green (seperti sekarang)
- "Custom" klik membuka date range picker di bawahnya

### 2. Date Range Picker (collapsible under "Custom")

```
Dari: [📅 2026-05-01] — Sampai: [📅 2026-05-31]   [Terapkan]
```

- Dua input date
- Shortcut pills: `7H` `30H` `90H` `1T` — klik langsung isi range

### 3. Periode Label In-Line

```
📊 Laporan
← 31 Mei – 6 Jun 2026 →     [Hari Ini] [Kemarin] [Minggu Ini] …
```

- Label di atas chips, dengan ← → untuk mundur/maju satu periode
- Contoh: di "Minggu Ini", ← mundur ke minggu lalu, → maju ke minggu depan
- Format label: "Hari ini, 1 Jun 2026" | "Kemarin, 31 Mei 2026" | "Minggu ini, 31 Mei – 6 Jun 2026" | "Bulan Juni 2026" | "Tahun 2026"

### 4. Periode Offset

Server support query `?period=week&offset=-1` (minggu lalu), `?period=month&offset=-2` (2 bulan lalu), dan `?date_from=2026-05-01&date_to=2026-05-31`

---

## Implementation Steps

### Step 1 — Server: `+page.server.ts`

- Tambah parse `date_from` + `date_to` dari URL params
- Tambah parse `offset` (int) — untuk navigasi ← →
- Jika `date_from`/`date_to` disediakan, gunakan itu langsung (meng-override `period`)
- Jika `period` + `offset`, kalkulasi maju/mundur:
  - `today` & `offset=-1` = kemarin
  - `week` & `offset=-1` = minggu lalu
  - `month` & `offset=-2` = 2 bulan lalu
  - `year` & `offset=0` = tahun ini
- Return `period` + `offset` + `dateFrom` + `dateTo` + computed `periodLabel`

### Step 2 — Frontend: `+page.svelte`

- **Preset chips**: `today`, `yesterday`, `week`, `month`, `last_month`, `year`, `custom`
- **Navigation ← →**: di samping periodLabel, dua tombol panah yang mengubah offset
- **Custom date inputs**: hidden by default, muncul saat preset = `custom`
- **Shortcut pills**: `7H` `30H` `90H` `1T` di bawah date range
- **Period label**: di header, format deskriptif

### Step 3 — Build & Verify

- Test semua 6 preset → data berbeda
- Test ← mundur, → maju → label berubah
- Test custom date range → data sesuai
- Test di mobile viewport → chips horizontal scroll

## Tradeoffs

| Pro | Con |
|-----|-----|
| 6 preset chips + custom = full flexibility | Sedikit lebih kompleks dari 3 tombol |
| ← → navigation intuitive | Pastikan offset tidak overshoot (tahun 1970) |
| Compact di mobile karena scrollable | Label panjang bisa wrap di mobile kecil |
| Custom date range = user bisa audit kapan aja | Date picker native HTML5 gap: tidak semua browser bagus |
