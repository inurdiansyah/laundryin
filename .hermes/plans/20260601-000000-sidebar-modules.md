# Rencana Implementasi Modul Sidebar (Non-POS)

Goal: Bangun UI + integrasi untuk 7 modul sidebar selain POS — Dashboard, Pesanan, Pelanggan, Inventori, Member, Pengantaran, Laporan, dan Pengaturan.

Tanggal: 2026-06-01

## Keadaan Saat Ini

| Modul | Route | Status | Keterangan |
|-------|-------|--------|------------|
| Dashboard | `/[slug]/` | Placeholder | KPI hardcoded, tidak fetch DB |
| Kasir (POS) | `/[slug]/pos` | **SELESAI** | Full UI + server load + action |
| Pesanan | `/[slug]/orders` | Placeholder | `<div>` kosong |
| Pelanggan | `/[slug]/customers` | Placeholder | `<div>` kosong |
| Inventori | `/[slug]/inventory` | Placeholder | `<div>` kosong |

| Pengantaran | `/[slug]/delivery` | Placeholder | `<div>` kosong |
| Laporan | `/[slug]/reports` | Placeholder | `<div>` kosong |
| Pengaturan | `/[slug]/settings` | Placeholder | `<div>` kosong |

Semua halaman belum punya `+page.server.ts` kecuali POS. DB schema sudah lengkap — semua tabel siap dipakai.

## Prioritas & Urutan

Urutan berdasarkan dependency dan nilai bisnis:

1. **Pesanan** — tabel `orders` + `order_items` + `order_status_log`
2. **Pelanggan** — tabel `customers`
3. **Dashboard** — agregasi dari orders + customers
4. **Inventori** — tabel `inventory_items` + `inventory_movements`
5. **Pengantaran** — tabel `drivers` + `delivery_schedules` + `delivery_routes`
6. **Member** — tabel `members` + `points_log`
7. **Laporan** — query agregasi lintas tabel
8. **Pengaturan** — tenant settings + layanan CRUD + user management

## Detail Per Modul

---

### Modul 1: Pesanan (`/[slug]/orders`)

**Tujuan:** Kelola semua order laundry — lihat, filter, update status, batalkan.

**Data dari DB:**
- `orders` JOIN `customers` (nama pelanggan)
- `order_items` (detail layanan per order)
- `payments` (pembayaran)

**Fitur UI:**
- Tabel/data list dengan: nomor order, pelanggan, total, status, waktu, aksi
- Filter: status (dropdown), tanggal (date range), pencarian nomor order/nama
- Status badge warna (11 status: diterima → selesai)
- Klik order → detail panel/modal: item list, payment info, status history
- Update status: dropdown/button next-status
- Batalkan order: konfirmasi modal
- Button "Proses" untuk advance status

**File:**
- `src/routes/(app)/[slug]/orders/+page.server.ts` — load + action (update status, cancel)
- `src/routes/(app)/[slug]/orders/+page.svelte` — UI

---

### Modul 2: Pelanggan (`/[slug]/customers`)

**Tujuan:** CRUD pelanggan — data dasar, riwayat order, total belanja.

**Data dari DB:**
- `customers` (semua kolom)
- Agregasi: jumlah order, total belanja

**Fitur UI:**
- Search input + tombol "Tambah Pelanggan"
- Tabel: nama, nomor HP, total belanja, jumlah order, terakhir transaksi
- Form tambah/edit: nama, nomor HP, alamat (modal atau side panel)
- Hapus: konfirmasi
- Klik pelanggan → riwayat order (filter orders by customer_id)

**File:**
- `src/routes/(app)/[slug]/customers/+page.server.ts`
- `src/routes/(app)/[slug]/customers/+page.svelte`

---

### Modul 3: Dashboard (`/[slug]/`)

**Tujuan:** Ringkasan bisnis — KPI real, order aktif, pelanggan terbaru.

**Agregasi dari DB:**
- Order hari ini: `count(*), sum(total)` WHERE `created_at::date = today`
- Order kemarin: untuk perbandingan naik/turun
- Order menunggu: count WHERE `status in ('diterima','menunggu_jemput')`
- Pelanggan baru minggu ini
- 5 order terbaru
- 5 pelanggan terbaru

**Fitur UI:**
- 3-4 KPI cards (order hari ini, pemasukan hari ini, order menunggu, pelanggan baru)
- Order aktif: tabel mini (nomor order, pelanggan, status, total)
- Pelanggan terbaru: list cards

**File:**
- `src/routes/(app)/[slug]/+page.server.ts`
- `src/routes/(app)/[slug]/+page.svelte` (ganti hardcoded)

## Konvensi UI

Semua modul mengikuti pola yang sama seperti POS:
- `rounded-xl border border-gray-200 bg-white` untuk card/panel
- Warna hijau (`green-600`) untuk aksi utama, biru untuk info, kuning/amber untuk warning
- Badge status warna (proses=biru, selesai=hijau, batal=merah)
- Toast/sukses pakai `enhance` + `applyAction` (seperti pola di POS)
- Search input dengan ikon 🔍 di kiri
- Mobile responsive: tabel scroll horizontal, form jadi full-width

## Validasi

Per modul:
1. `npm run build` — harus zero error
2. UI render di browser — data termuat
3. Test create + update + delete (kalau ada)

## Risiko

- Semua data harus di-filter per `tenant_id` (RLS sudah ada dari migration)
- Query `customer` di server load perlu JOIN untuk total belanja (agregasi dari orders)
- Modul Pengantaran paling kompleks — perlu 3 sub-view + relationship ke orders
- Format Rupiah sudah ada di `$lib/utils/format.ts` → `formatRupiah()`

## Catatan

- `+page.server.ts` pattern: `load()` return data, `actions` handle POST
- Semua load pakai Supabase server client via `getServerSupabase()`
- Tenant ID dari `locals.tenant.tenant_id` (hooks)
- Tidak perlu migration schema baru — semua tabel sudah ada
