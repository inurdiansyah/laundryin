# Rencana Perbaikan UX & Bug — Evaluasi LaundryIn

Goal: Memperbaiki 7 masalah UX, bug, dan workflow gap berdasarkan evaluasi seed data 1 bulan.

---

## 1. Dashboard: KPI Cards Clickable → Navigasi

**Problem:** KPI cards (Order Hari Ini, Pemasukan, Menunggu Diproses, Pelanggan Baru) statis — user tidak bisa klik untuk drill-down.

**Fix — `/src/routes/(app)/[slug]/+page.svelte`**:
- Bungkus setiap card dengan `<a href="/{slug}/orders">` atau `<a href="/{slug}/customers">` dsb
- **Order Hari Ini** → halaman Pesanan
- **Pemasukan Hari Ini** → halaman Laporan
- **Menunggu Diproses** → halaman Pesanan dengan filter `status=diterima`
- **Pelanggan Baru** → halaman Pelanggan
- Order Aktif items → klik langsung ke halaman Pesanan

**No server changes needed.**

---

## 2. Pesanan: Pagination

**Problem:** Server load `.limit(50)`, UI tidak ada offset/page control. Lebih dari 50 order tidak bisa diakses.

**Fix — `orders/+page.server.ts`**:
- Tambah query param `page` (default 1)
- Gunakan Supabase `range((page-1)*20, page*20 - 1)` untuk pagination
- Return `{ orders, totalCount, page, perPage: 20 }`

**Fix — `orders/+page.svelte`**:
- Tambah pagination UI di bawah list: `< Sebelumnya | Halaman 1/3 | Selanjutnya >`
- Link pagination ke URL search params (SSR-friendly, bukan client-side)

---

## 3. POS Flow: Auto-Scroll + Jemput → Delivery Schedule

**Problem A:** Setelah submit order berhasil, toast muncul di atas tapi viewport tetap di bawah (cart panel). User harus scroll manual ke atas.

**Fix — `pos/+page.svelte`**:
- Setelah `applyAction` sukses, jalankan `window.scrollTo({ top: 0, behavior: 'smooth' })`
- Tambahkan juga scroll ke toast di mobile view

**Problem B:** Pilih jalur "Jemput" → status langsung `menunggu_jemput`, tapi tidak ada delivery schedule yang dibuat. Di halaman Pengantaran, pesanan jemput tidak muncul.

**Fix — `pos/+page.server.ts`**:
- Di action `default`, setelah insert order:
  - Jika `jalur === 'jemput'` → set `status = 'menunggu_jemput'` (bukan 'diterima')
  - **Auto-create delivery schedule**: insert ke `delivery_schedules` dengan:
    - `tipe = 'jemput'`, `status = 'terjadwal'`
    - `tanggal = today` (bisa di-override via picker nanti)
    - `slot_waktu = 'pagi'` (default)
    - `alamat` dari customer
    - `driver_id = null` (menunggu assign operator)

**File:** `pos/+page.server.ts` baris 134-148 (insert order) + setelah line 168 (insert status log)

## 4. Pengantaran: Filter + UX Redesign

**Problem:** 
- Tidak ada filter status (semua jadwal campur: selesai, terjadwal, pending)
- Driver list tidak terlihat jelas (hanya inline dropdown per jadwal)
- UX kurang intuitif untuk operator lapangan

**Fix — `delivery/+page.server.ts`**:
- Support query params: `status` (terjadwal/driver_berangkat/dijemput/tiba_di_laundry/selesai), `tipe` (jemput/antar)
- Return juga semua driver (termasuk tidak aktif) untuk management

**Fix — `delivery/+page.svelte`**:
- **Tab filter di atas**: "Semua | Menunggu Jemput | Sedang Jemput | Sedang Antar | Selesai"
  - Terjemahan UX: `terjadwal` → "Menunggu Jemput", `driver_berangkat`+`dijemput`+`tiba_di_laundry` → "Sedang Jemput", antar yang belum selesai → "Sedang Antar", `selesai` → "Selesai"
- **Driver panel sidebar kanan** (mobile: modal): list semua driver + status mereka
- **Card redesign** untuk lebih scan-friendly:
  - Warna background per tipe: jemput = biru muda, antar = hijau muda
  - Large status icon di kiri
  - Nomor order + customer name bold
  - Alamat + slot waktu jelas terbaca
  - Driver assigment: chip dengan nama driver (bukan dropdown aneh)

**No schema changes.**

## 5. Member: Pagination + Filter Tier

**Problem:** Semua member di-load sekaligus, tidak ada pagination, tidak ada filter tier.

**Fix — `members/+page.server.ts`**:
- Support query params: `page`, `tier`
- Pagination: `range()` + return `totalCount`
- Return `{ members, totalCount, page, perPage: 15 }`

**Fix — `members/+page.svelte`**:
- **Tab tier filter**: "Semua | Regular | Silver | Gold | Platinum"
- Pagination UI di bawah list
- Search by nama member atau nomor member

## 6. Laporan: Top 10 Layanan Kosong

**Problem:** Query `order_items` join tidak tepat — `order_items` tidak punya `created_at`, join via `orders` dengan filter `orders.tenant_id` gagal. Data top services selalu kosong.

**Fix — `reports/+page.server.ts`**:
- Ganti query order_items: join via orders
  ```sql
  supabase.from('order_items')
    .select('nama_layanan, subtotal, orders!inner(tenant_id, created_at)')
    .eq('orders.tenant_id', tenantId)
    .gte('orders.created_at', monthStart)
  ```
- Atau alternatif: gunakan subquery / RPC
- Verifikasi dengan seed data: harusnya "Cuci + Setrika Reguler" di posisi #1

## 7. Inventori: Workflow Operator (Daily Stock Reporting)

**Problem:** Operator tidak bisa melaporkan stok harian dengan mudah. Saat ini harus input stok masuk/keluar satu per satu — tidak praktis untuk item seperti deterjen dan plastik yang dipakai setiap hari.

**Rekomendasi workflow baru:**

**Konsep "Stok Opname Harian"** — operator cukup input stok aktual hari ini, sistem hitung selisih otomatis:

**Fix — `inventory/+page.svelte`**:
- Tambahkan tombol **"📋 Opname Harian"** yang membuka modal/form khusus
- Form opname: tabel semua item dengan kolom:
  - Nama item (readonly)
  - Stok sistem saat ini (readonly, abu-abu)
  - **Stok Aktual** (input — operator hitung fisik)
  - Selisih (auto-hitung: aktual - sistem)
- Submit → untuk setiap item dengan selisih, auto-create `movement`:
  - Selisih positif → `tipe: 'masuk'`, keterangan: "Opname harian DD/MM"
  - Selisih negatif → `tipe: 'keluar'`, keterangan: "Opname harian DD/MM — pemakaian"
  - Selisih 0 → skip (tidak ada mutasi)
- Simpan timestamp → satu klik lapor semua

**Fix — `inventory/+page.server.ts`**:
- Action `daily_opname`: terima array `[{item_id, stok_aktual}]`, hitung selisih per item, insert movements, update stok

**Kenapa ini lebih baik:**
- Operator hanya input angka akhir (yang mereka lihat di rak), bukan hitung pemakaian
- 6 item × hitung fisik = 2 menit, vs input masuk/keluar bisa 10+ menit
- Selisih otomatis jadi report pemakaian harian
- Cocok untuk laundry: "deterjen habis berapa? lihat selisih opname"
